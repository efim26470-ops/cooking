# Offline Cookbook

Статическое PWA-приложение для iPhone, iPad, macOS и Windows. Сервер и база данных не требуются: рецепты, фотографии, остатки и покупки хранятся в браузере пользователя.

## Возможности

- встроенный русскоязычный каталог рецептов;
- собственные рецепты и локальные фотографии;
- автоматический пересчёт ингредиентов по числу порций;
- пошаговый полноэкранный режим с блокировкой сна экрана;
- несколько параллельных кухонных таймеров и уведомления;
- список покупок с объединением одинаковых ингредиентов;
- учёт домашних остатков и сроков годности;
- подбор блюд по доступным продуктам и процент совпадения;
- категории для аэрогриля, духовки, мультиварки и плиты;
- избранное, поиск, фильтры и темы оформления;
- импорт/экспорт всей книги в JSON;
- опциональный импорт открытого каталога TheMealDB с фотографиями, источниками и YouTube-ссылками;
- офлайн-режим через Service Worker.

## Публикация на GitHub Pages

1. Создайте новый репозиторий.
2. Загрузите **содержимое** этой папки в корень репозитория.
3. Откройте `Settings → Pages`.
4. Выберите `Deploy from a branch`, ветку `main`, папку `/root`.
5. После публикации откройте адрес сайта в Safari.

## Установка на iPhone

1. Откройте опубликованный сайт именно в Safari.
2. Нажмите «Поделиться».
3. Выберите «На экран Домой».
4. Запустите приложение с созданной иконки хотя бы один раз при наличии интернета, чтобы оболочка закэшировалась.

## Хранение данных

Основные данные хранятся в IndexedDB. Таймеры сохраняются в localStorage. Фотографии перед сохранением уменьшаются и переводятся в JPEG, чтобы не расходовать лишнее место. Очистка данных Safari или удаление данных сайта удалит локальную книгу, поэтому периодически используйте экспорт JSON.

## Внешний каталог

Кнопка `Настройки → Загрузить открытый каталог` обращается к публичному API TheMealDB. Импортированные записи сохраняют ссылки на первоисточник и видео. Для коммерческой публикации отдельно проверьте условия использования API, изображений и исходных сайтов.
