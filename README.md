# Offline Cookbook 4.7

## Исправление перевода

Версия 4.3 полностью убирает пословную транслитерацию вида «тхе», «меанвхиле», «коокинг». При загрузке интернет-каталога приложение использует качественный машинный перевод, сохраняет исходный английский текст и кэширует уже переведённые карточки в локальной базе. Если внешний переводчик временно недоступен, приложение оставляет непереведённый фрагмент на английском вместо бессмысленной кириллической транслитерации.

После обновления старые повреждённые переводы обнаруживаются автоматически. При наличии интернета каталог повторно загружается из первоисточников и заменяет испорченные карточки, сохраняя избранное и пользовательские данные.


Статическое PWA-приложение для iPhone, iPad, macOS и Windows. Все пользовательские данные хранятся локально в IndexedDB. Проект готов к публикации через GitHub Pages.

## Исправления версии 4.1

- карточки рецептов теперь открываются обычным касанием на iPhone, iPad и в установленном PWA;
- добавлены прямые обработчики касаний, клавиатурное управление и полноценные интерактивные состояния карточек;
- исправлено открытие старых рецептов после обновления с предыдущих версий;
- повреждённые или устаревшие структуры ингредиентов и шагов восстанавливаются автоматически;
- добавлена заметная кнопка закрытия карточки рецепта;
- исправлены обрезание длинных названий, переполнение метаданных и поведение модального окна;
- Service Worker обновлён до сетевой стратегии для файлов интерфейса, поэтому старый JavaScript больше не должен оставаться в кэше.

## Как работает автозагрузка

После запуска приложение проверяет дату последней синхронизации. Если каталог ещё не загружался, содержит мало интернет-рецептов или старше 7 дней, обновление запускается в фоне. В разделе `Ещё` можно отключить автообновление или запустить обновление вручную.

Перевод выполняется непосредственно в браузере без внешнего ключа API. Поэтому приложение остаётся полностью статическим и совместимым с GitHub Pages. Если онлайн-переводчик временно недоступен, редкий непереведённый фрагмент остаётся на английском; бессмысленная кириллическая транслитерация больше не используется. Оригинальное название всегда сохраняется.

## Публикация на GitHub Pages

1. Полностью замените старые файлы содержимым папки проекта в корне репозитория.
2. Откройте `Settings → Pages`.
3. Выберите `Deploy from a branch`, ветку `main`, папку `/root`.
4. После публикации закройте старую вкладку/PWA и откройте сайт снова. Версия 4.3 автоматически обновит Service Worker, обнаружит старые повреждённые переводы и запустит их восстановление.
5. Дождитесь завершения первой фоновой синхронизации каталога.

## Установка на iPhone

Откройте сайт в Safari → `Поделиться` → `На экран Домой`.

## Важно

Очистка данных Safari удалит локальную базу. Периодически сохраняйте экспорт JSON. Перед коммерческим использованием проверьте условия использования внешних источников и изображений.


## Обновление каталога

Откройте «Ещё» → «Обновить каталог и перевод». Кнопка немедленно покажет текущий этап. Если фоновая синхронизация уже идёт, откроется её живой прогресс.
